람다식, Lambda Expression
- 함수형 프로그래밍 스타일을 지원
- 자바의 메소드 표현과 유사
- 람다식이란 이름이 어려워 보이지만 메서드를 하나의 식(expression)으로 표현한 것을 뜻합니다

- 자바의 메소드를 좀 더 쉽고 간결하게 정의하기 위한 기법



람다식 기본 표현(문법)
- (자료형 매개변수) -> { 실행 코드;}
- a. 매개변수 : 메소드의 매개변수와 동일한 역할
- b. -> : 화살표(Arrow), 코드블럭을 호출하는 역할
- c. 실행코드: 메소드의 구현부와 동일한 역할



- public void test(int a){
	syso(a);
}
-function test(){}
- function(){}
- () {}


자바의 람다식은 불완전하다. -> 자바는 객체 지향이라 메소드가 독립적이지 않다. 자바는 메소드를 객체 취급할 수 없다.
> 자바의 람다식은 인터페이스를 사용해서(도움을 받아서) 정의한다.
